# VPN

## クライアントVPN

### 認証タイプ

- Active Directry → Managed Microsoft AD , AD Connector
- シングルサインオン → SAMLフェデレーションでIAM SAML IDプロバイダー作成しクライアントVPNエンドポイントの認証
- 相互認証 → ACMにサーバー証明書とクライアント証明書をアップしして使用

## site-to-site VPN

- ASN → 1つのネットワークのまとまりを表す識別子
- BGP → AS同士がネット経路情報を交換するためのルーティングプロトコル  

- VPNでつなぐときはそれぞれのASNで経路を決める
- BGPが有効 → 動的ルーティング
- 無効 → 静的ルーティング

### VPN接続
#### 機能
- Internet Key Exchange バージョン2(IKEv2) → 暗号化のための共通鍵を交換する
- NATトラバーサル(NAT-T) → オンプレミス側にNATルーターを介した接続
- デッドピア検出(DPD) → 接続先のデバイスが有効化確認 [タイムアウトしたときに接続を クリア、再起動、何もしない を選べる]  

- デフォルトでは仮想プライベートゲートウェイ(VGW)は2つのAZにトンネルを置きそれぞれにパブリックIPを付与
→カスタマーゲートウェイ(CGW)でそのトンネル二つを指定して冗長化
- IKEネゴシエーションの場合は逆も可能  

1つのVGWに複数のCGWをつなぐ→VPN CloudHUb

#### ソフトウェアVPN
- コンプライアンス要件により両端を完全に制御する必要があるときはEC2にソフトウェアVPNをセットアップしてIGW経由でVPNもできる
