# その他Codeサービス
## Cloud9
- ブラウザでIDEを実行できクラウドを通してチーム開発できる

## CodeGuru
- ソースコードのレビューでバグや問題の抽出、パフォーマンス最適化の自動化

## CodeStar
- Codeサービスを組み合わせCI/CDパイプラインをテンプレートから構築してすぐ開発できる

## CodeArtifact
- パッケージを適切なアクセス権でチームに公開、共有できる

## OpsWorks
- Chef、Puppetの機能をマネージドで提供しデプロイをコントロールできる

# CodeCommit
- Gitと同じサービスをチームに提供
- IAMポリシーでアクセス制御できる
- KMSのawsマネージドキーで暗号化されてる
- CloudTrailで記録

# CodeBuild
- ビルドとテストを実行するコンテナ環境をマネージドで提供
- buildspec.ymlに記載のコマンドでテスト、コンパイル、パッケージ作成、コンテナイメージの作成など実行
- 作成されたパッケージはS3やbuildspec.ymlに定義したdocker pushコマンドでアップロードした
ECRリポジトリに保存される

# CodeDeploy 
- EC2、ECS、Lamdaやオンプレミスサーバーへのデプロイを自動化
- アプリケーションリビジョン(アプリをインストールするためのファイル群)
の保存先にS3やGitHubを指定できる

1. ec2、オンプレ、Lamda、ECSどれにするか決定
2. 対象の上記を選択
3. リビジョン(S3かGitHub)、AppSpec(アプリケーション仕様)→デプロイ処理詳細、デプロイ設定(方法)を指定
※ EC2やオンプレにはCodeDeployエージェントをインストール

## AppSpec
- デプロイ仕様
- YAMLで記述した通りに実行される

# デプロイ設定
## EC2インスタンス/オンプレミス
### AllAtOnce
- 一度にすべてのインスタンスにデプロイ
- 正常ホスト 0

### HalfAtATime
- 一度に最大半分のインスタンスにデプロイ
- 正常ホスト 50%<=

### OneAtATime 
- 一台ずつデプロイ
- 正常ホスト 1<=

## ECS
(例)
ECSCanary10Percent5Minutes- 最初10% → 5分後に90%
ECSLinear10PercentEvery1minutes - 最初10% → 1分ごとに10%ずつ移行
ECSAllAtOnce - 一度にすべて

## Lamda
バージョンとエイリアスの紐づけ
Lamda10Percent5Minutes- 最初10% → 5分後に90%
LamdaLinear10PercentEvery1minutes - 最初10% → 1分ごとに10%ずつ移行
LamdaAllAtOnce - 一度にすべて
